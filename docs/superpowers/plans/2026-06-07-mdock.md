# mdock Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn margo's bar-only dock into **mdock** — one dock core with two display modes (bar pill + standalone per-output layer-shell surface that can be always-on, auto-hide, or toggle), fully Settings-driven, porting hydock's UX onto margo IPC.

**Architecture:** The dock *strip* already exists as the relm4 `MargoDockModel` (a `DynamicBox` factory of `MargoDockItemModel` icons, fed by `margo_service().clients` + `pinned_apps_store`). We (1) grow the `Dock` config with the new knobs + two enums, (2) add pure position/behavior→layer-shell mapping helpers, (3) build a **standalone surface** component that hosts a `MargoDockModel` inside its own layer-shell window with Always/AutoHide/Toggle behavior, (4) wire `mshellctl dock` IPC + a startup/teardown hook, (5) grow the Settings page and brand everything "mdock", (6) fix the parked click→focus-by-`Address` bug, (7) add a hover preview card.

**Tech Stack:** Rust, relm4/GTK4, gtk4-layer-shell, reactive_stores, zbus (mshell IPC), margo socket IPC via `mshell-margo-client`.

**Key existing pieces (reuse, don't reinvent):**
- `mshell-crates/mshell-frame/src/bars/bar_widgets/margo_dock.rs` — `MargoDockModel` (the strip). 493 lines.
- `mshell-crates/mshell-frame/src/bars/bar_widgets/margo_dock_item.rs` — `MargoDockItemModel` (per-app icon, click/middle/scroll/menu). 889 lines.
- `mshell-cache::pinned_apps::pinned_apps_store` — **pins already live here** (NOT config). Keep it as the pin source of truth.
- `margo_service().clients` (reactive) — running windows; `Address`; `focuswindow` dispatch.
- `mshell-crates/mshell-frame/src/frame.rs` — layer-shell surface pattern (`init_layer_shell`, `Layer`, `set_anchor`, `auto_exclusive_zone_enable`, `Revealer` reveal). Model for the standalone surface.
- `mshell-crates/mshell-settings/src/dock_settings.rs` — the page to grow. 268 lines.
- IPC pattern: `mshell-core/src/ipc.rs` (`#[interface] com.mshell.Shell`) + `mshellctl/src/subcommands/*` + `mshellctl/src/app.rs` + `mshellctl/src/main.rs`. (See the just-shipped `log` subcommand as a template.)

**Branding:** user-facing strings, the `mshellctl dock` verb, Settings title, and docs say **"mdock"**. Internal Rust identifiers (`Dock`, `MargoDock*`, `dock_*`) and the serde config key `dock` stay as-is (no config break, minimal churn).

```rust
//! Pure mapping from mdock config enums to gtk4-layer-shell geometry.
//! No GTK widgets here — just the rules, so they're unit-testable.

use gtk4_layer_shell::Edge;
use mshell_config::schema::config::{DockBehavior, DockPosition};
use relm4::gtk::Orientation;

/// The screen edge the standalone dock anchors to.
pub(crate) fn edge_for(p: DockPosition) -> Edge {
    match p {
        DockPosition::Top => Edge::Top,
        DockPosition::Bottom => Edge::Bottom,
        DockPosition::Left => Edge::Left,
        DockPosition::Right => Edge::Right,
    }
}

/// The dock strip's box orientation for a given edge.
pub(crate) fn orientation_for(p: DockPosition) -> Orientation {
    match p {
        DockPosition::Top | DockPosition::Bottom => Orientation::Horizontal,
        DockPosition::Left | DockPosition::Right => Orientation::Vertical,
    }
}

/// Only an always-on dock reserves an exclusive zone (tiled windows shrink
/// to make room). Auto-hide / toggle float over the windows.
pub(crate) fn reserves_exclusive_zone(b: DockBehavior) -> bool {
    matches!(b, DockBehavior::Always)
}

/// Only auto-hide needs the thin edge trigger surface.
pub(crate) fn uses_edge_trigger(b: DockBehavior) -> bool {
    matches!(b, DockBehavior::AutoHide)
}
```

```rust
/// Visible app classes for the dock: running apps minus the ignore list,
/// lower-cased compare. Pinned apps are merged in by the existing
/// pinned_apps_store path; this only governs the *running* additions.
pub(crate) fn visible_running_classes(
    running: &[String],
    ignore: &[String],
    show_running: bool,
) -> Vec<String> {
    if !show_running {
        return Vec::new();
    }
    let ignore_lc: Vec<String> = ignore.iter().map(|s| s.to_lowercase()).collect();
    running
        .iter()
        .filter(|c| !ignore_lc.contains(&c.to_lowercase()))
        .cloned()
        .collect()
}
```

```rust
impl Component for MdockSurface {
    type CommandOutput = ();
    type Input = MdockSurfaceInput;
    type Output = ();
    type Init = MdockSurfaceInit;
    type Root = gtk::Window;
    type Widgets = ();

    fn init_root() -> Self::Root {
        gtk::Window::new()
    }

    fn init(_: Self::Init, window: Self::Root, _sender: ComponentSender<Self>) -> ComponentParts<Self> {
        let cfg = config_manager().config().dock().get_untracked();

        window.init_layer_shell();
        window.set_namespace(Some("mdock"));
        window.set_layer(Layer::Top);
        window.set_keyboard_mode(gtk4_layer_shell::KeyboardMode::None);
        let edge = edge_for(cfg.position);
        window.set_anchor(edge, true);
        if reserves_exclusive_zone(cfg.behavior) {
            window.auto_exclusive_zone_enable();
        }

        let dock = MargoDockModel::builder()
            .launch(MargoDockInit {
                orientation: orientation_for(cfg.position),
                // mirror the existing Init fields (read margo_dock.rs); pass
                // icon_size/spacing from cfg.
                ..mdock_init_defaults(&cfg)
            })
            .detach();

        let revealer = gtk::Revealer::builder()
            .transition_type(gtk::RevealerTransitionType::Crossfade)
            .transition_duration(180)
            .child(dock.widget())
            .reveal_child(matches!(cfg.behavior, DockBehavior::Always))
            .build();
        revealer.add_css_class("mdock-surface");
        window.set_child(Some(&revealer));

        if matches!(cfg.behavior, DockBehavior::Always) {
            window.set_visible(true);
        }

        ComponentParts {
            model: MdockSurface { revealer, window, _dock: dock, behavior: cfg.behavior },
            widgets: (),
        }
    }

    fn update(&mut self, msg: Self::Input, _sender: ComponentSender<Self>, _root: &Self::Root) {
        match msg {
            MdockSurfaceInput::Show => { self.window.set_visible(true); self.revealer.set_reveal_child(true); }
            MdockSurfaceInput::Hide => {
                self.revealer.set_reveal_child(false);
                if !matches!(self.behavior, DockBehavior::Always) { self.window.set_visible(false); }
            }
            MdockSurfaceInput::Toggle => {
                let now = self.revealer.reveals_child();
                self.update(if now { MdockSurfaceInput::Hide } else { MdockSurfaceInput::Show }, _sender, _root);
            }
        }
    }
}
```

```rust
// inside init(), after the dock window is set up:
if uses_edge_trigger(cfg.behavior) {
    let trigger = gtk::Window::new();
    trigger.init_layer_shell();
    trigger.set_namespace(Some("mdock-trigger"));
    trigger.set_layer(Layer::Top);
    trigger.set_anchor(edge, true);
```

```rust
mod focus_tests {
    /// Given the addresses of an app's windows and the last-focused one,
    /// the next click targets the next address (wrapping).
    fn next_address(addrs: &[&str], last: Option<&str>) -> Option<String> {
        if addrs.is_empty() { return None; }
        let idx = last.and_then(|l| addrs.iter().position(|a| *a == l));
        let next = match idx { Some(i) => (i + 1) % addrs.len(), None => 0 };
        Some(addrs[next].to_string())
    }
    #[test]
    fn click_cycles_then_wraps() {
        let a = ["0x1", "0x2", "0x3"];
        assert_eq!(next_address(&a, None).as_deref(), Some("0x1"));
        assert_eq!(next_address(&a, Some("0x1")).as_deref(), Some("0x2"));
        assert_eq!(next_address(&a, Some("0x3")).as_deref(), Some("0x1"));
    }
}
```
