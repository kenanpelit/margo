# mplay Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A first-party `mplay` binary that controls an mpv window through margo (`mctl` + mpv JSON IPC) and plays native video wallpapers (in-tree mpvpaper port), replacing `margo-mpv.sh`.

**Architecture:** New workspace binary crate `mplay/`. A small **controller** (clap subcommands → `mctl` subprocess + mpv JSON-IPC unix socket) and a larger **wallpaper engine** (`paper/`: wlr-layer-shell background surface + EGL + hand-written libmpv render-gl FFI, driven by a calloop loop). Pure logic (corner math, URL detection, JSON→rect mapping, option strings) is split into testable modules; the engine is verified manually.

**Tech Stack:** Rust 2024, clap, serde_json, `wayland-client 0.31`, `wayland-protocols-wlr 0.3`, `khronos-egl 6`, `wayland-egl`, `calloop`, hand-written `extern "C"` libmpv FFI (`-lmpv`).
