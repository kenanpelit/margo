## margo v1.0.0

Rust-native Wayland compositor + GTK4/relm4 desktop shell, built on Smithay.
This is the 1.0 milestone — the historical C codebase is fully replaced by the
Rust workspace. Below is everything that landed since **v0.9.9**.

### ✨ New features
- **Settings → Backup** — config profiles plus whole-config export / import /
  reset. Bundles the shell profiles and the compositor's `config.conf` +
  `binds.conf` into a portable `.tar.gz`; import restores through dotfiles
  symlinks and reloads live.
- **Settings → Startup** — a consolidated page with a full autostart-script
  manager. Autostart now accepts **arbitrary commands** with a per-entry delay
  and an Every-start / Login-only trigger, run in a transient systemd scope.
- **Sound → Startup defaults** — default output + input volume restored at each
  login (PipeWire forgets levels across reboots). Opt-in switch + two sliders,
  applied via `wpctl` once WirePlumber settles.
- **Run-or-raise** — new `focusapp` action and `cycle` summon mode that walks
  through matching windows, plus `and` / `or` / `difference` window-match
  operators.
- **Settings → Input** — gesture bindings are now editable in place.
- **Surface opacity** — ashell-style frosting for the shell surfaces, with the
  design language documented in DESIGN.md.
- **Bar** — custom-widget `exec` can emit a `#state` CSS class for live theming.
- **Settings search** — a flat results list; sub-pages are now findable by name.
- **Weather** — shows the last cached reading when the provider is unreachable.

### 🎨 Design
- **One button system** across every menu, widget and plugin: menu action
  buttons (power, session, DNS/VPN, network, valent) are one slim fully-rounded
  pill family (`.ok-button-cell`), and plugin chips adopt the tile shape
  language so selectors read consistently. Tall toggle tiles (control center)
  stay their own kind; the power charge-limit + DNS action buttons match the
  control-center power-profile segmented buttons.
- GNOME/ashell-style rounding pass: softer shape scale, very rounded
  control-center tiles, rounder bar pills + window defaults, comfier button
  padding, fully-rounded dashboard cards.
- Sound volume + default-level sliders are proper coloured bars (filled track +
  knob + `NN%`), all equal width.

### 🐛 Fixes
- **Fresh-login bar stall** — the bar could take ~30s to appear after login
  (wallpaper visible, no bar). The weather Hourly widget launched a full relm4
  component **per forecast hour** (each with two reactive config watchers),
  built eagerly per output — hundreds of component launches on the GTK main
  thread that burned ~15s at 100% CPU before the bar could paint. The cells are
  now plain widgets with the unit/format watch owned centrally.
- **DNS/VPN menu could freeze the whole shell** when toggling Blocky: the
  privileged action fell back to `pkexec`, whose polkit password dialog can't
  take keyboard focus behind the menu's exclusive layer-shell grab, so it hung
  forever. Privileged actions are now strictly non-interactive (passwordless
  `sudo -n` / askpass, else a toast — never `pkexec`), toggles are serialised,
  and the whole VPN/DNS flow is modelled on the `osc-mullvad` script: Mullvad
  and Blocky are mutually exclusive (Blocky is the no-VPN DNS fallback) and
  every transition fires a desktop notification.
- Autostart scripts survive an `mshell` restart (transient scope, detached from
  mshell's cgroup) instead of being reaped.
- Screenshot region selector takes keyboard focus over an open menu, so Enter
  commits the region.
- Settings: sub-pages reachable from the sidebar search; list-editor rows no
  longer widen the panel; clearer `$PATH` status on the Startup script list.
- Compositor: graceful `focus_under` under session lock (no panic); resolve the
  `XF86AudioPlayPause` keysym (xkbcommon gap).
- Weather: back off 1 h on a rate-limit instead of fast-retrying.

### 🔧 Internals
- Settings refactor: table-driven sidebar / content-stack / search registry,
  guarded with tests.
- `RepositionMenus` collapsed into a macro.
- New unit-test coverage: core IPC (clipboard verbs), portal screencast picker,
  matugen hex + audio device pick, margo-client `state.json` parse, settings +
  weather pure functions.
