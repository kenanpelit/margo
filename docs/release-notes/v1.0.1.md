# margo v1.0.1

A maintenance + feature release on top of 1.0.0. Headline: the **mdock**
dual-mode dock, **file logging** for both the compositor and the shell, and a
calmer **weather** poller.

## mdock — dual-mode dock

The dock is now branded **mdock** and runs in two complementary modes off one
core strip:

- **Bar pill** (unchanged) — per-app icons (pinned + running, with a window
  count), click-to-focus, hover preview, ignore list.
- **Standalone** — a per-output dock with two styles, selectable in
  **Settings → Widgets → mdock**:
  - **Layer-shell (bar-attached)** — the dock opens *inside* the bar's frame,
    exactly like the session menu. `Esc` / click-away close it; the launcher
    button opens the app launcher.
  - **Popup (independent)** — the dock is its own floating layer-shell window,
    honouring **Always / Auto-hide** (1 px edge trigger) **/ Toggle** and a
    configurable **edge** (top / bottom / left / right).

Controlled by `mshellctl dock toggle | show | hide`. Every knob — modes,
behaviour, position, ignore list, hover preview, launcher command, spacing,
icon overrides — lives on the dedicated Settings page.

## File logging — margo + mshell

Both the compositor and the shell now write durable, rotating log files under
`~/.local/state/margo/logs/` (the last **3 sessions** are kept; each start gets
its own `{app}-YYYYMMDD-HHMMSS.log` plus a `{app}-latest.log` symlink). Log
levels are adjustable **live**:

- Compositor: `.conf` knobs (`log_to_file`, `log_file_level`,
  `log_keep_sessions`) + `mctl log level|enable|disable|path|open`.
- Shell: `[logging]` profile section + `mshellctl log …` + **Settings →
  Logging**.

The file sink follows the configured level, independent of `RUST_LOG` /
`MARGO_LOG` (those still steer stdout).

## Weather — gentle backoff

On any fetch failure the weather poller now backs off to a single
**configurable retry interval** (default **60 min**, settable up to 24 h in
**Settings → Widgets → Weather**) instead of hammering the API every couple of
minutes on a rate-limit. The poll interval is now only re-armed when it actually
changes, so the backoff timer truly elapses (previously every error re-started
the poller and re-fetched immediately, defeating the backoff).

## Settings polish

Action buttons that previously read as plain text (Open log folder, Add
bindings, Refresh, Browse, Preview, …) are now outlined / elevated so they
actually look like buttons.
